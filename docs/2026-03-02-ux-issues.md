# UX Issues Log — 2026-03-02

Running log of UX issues identified during live review testing. Each item noted during field-by-field review of foreclosure entity.

## Issues Found

1. **Batch generation panel visible to all users** — should be on a separate admin-only page. Reviewers don't need to see batch run history or trigger generation. Needs role-based routing.
2. **`actual_sale_date` review not marked accepted** — Submit button wasn't saving status before navigation fix was applied. Fixed mid-session.
3. **No validation gate on corrections** — reviewer verdicts flow directly into Entity Knowledge without validation. A mistaken correction immediately degrades future generations. Need a `pending_validation` → `validated` workflow before corrections are applied.
4. **No question curation pipeline** — AI/reviewer-generated questions go straight to open status. Need a `draft` → `approved` pipeline with dedup before presenting to ServiceMac.
5. **No way to promote chat answers to question resolutions** — useful answers from AI Assistant chat stay in the session, not connected to the question system.
6. **Duplicate learnings created** — multiple clicks on the same verdict create duplicate learning records. Need idempotency check.

## Issues Found (2026-03-03)

7. **Atlas name confusing** — "Atlas" was originally a knowledge graph concept but the feature is just BigQuery data preview. Renamed display label to "Data Preview". Internal route still `/atlas` — full rename deferred.
8. **No context traceability** — reviewers couldn't see which context docs the model used for a mapping. No way to verify "per Entity Knowledge corrections" claims. Fixed: implemented full citation + inline viewer system (see session log).
9. **AI review text not rendered as markdown** — pre-generated reviews displayed as `whitespace-pre-wrap` plain text, missing formatting. Fixed: now uses `CitationMarkdown` with full markdown + citation link rendering.
