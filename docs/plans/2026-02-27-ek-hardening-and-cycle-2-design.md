## Task 1: Strengthen EK Phrasing (4 files)

### 1a. `scripts/auto-verdicts.ts` — Harden note templates (lines 84-103)

| Match type | Current | New |
|---|---|---|
| `NO_GEN` | `"Should map to: X"` | `"REQUIRED: Map to X. This is a verified correction."` |
| `DISJOINT` | `"Should be: X. Currently mapped to: Y"` | `"REQUIRED: Use X (not Y). This is a verified correction — do not override."` |
| `OVERLAP/SUBSET` | `"Expected sources: X. Generated: Y"` | `"REQUIRED: Must include all of: X (currently only: Y). Verified correction."` |
| `SUPERSET` | `"Has extra sources... Expected: X. Generated: Y"` | `"REQUIRED: Use only X (not Y). Extra sources are incorrect. Verified correction."` |

### 1b. `src/lib/generation/mapping-learning.ts` — Harden verdict learning content (lines 305-310)

| Verdict | Current | New |
|---|---|---|
| `wrong_table` | `"Source table is wrong. Model used X."` | `"CORRECTION (MANDATORY): Wrong source table. Do NOT use X."` |
| `wrong_field` | `"Source field is wrong within X."` | `"CORRECTION (MANDATORY): Wrong source field within X."` |
| `missing_source` | `"This field has a source but was left unmapped."` | `"CORRECTION (MANDATORY): This field MUST be mapped — do not leave unmapped."` |
| `should_be_unmapped` | (already strong) | No change |

```
or override these corrections under any circumstances.
```

## Task 1.5: Manual Feedback Capture Validation

Before running the full cycle, manually test the feedback pipeline with a handful of fields to verify:

1. **Source verdict feedback** — give a `wrong_table` and `wrong_field` verdict via the UI or API, confirm the learning record is created with the new MANDATORY phrasing, confirm Entity Knowledge is rebuilt with the correction
2. **Transform verdict feedback** — give a `wrong_enum` or `wrong_logic` verdict, confirm learning and EK rebuild
3. **Question feedback** — answer a structured question generated by the model (resolve it with an answer), confirm it appears in EK under "Resolved Questions"

### Suggested test fields

Pick 3-4 fields from an entity with existing mappings (e.g., `foreclosure` or `bankruptcy_case`):
- Field A: Give `wrong_table` verdict with notes pointing to a different source table
- Field B: Give `wrong_field` verdict with notes pointing to a different column
- Field C: Give `wrong_logic` transform verdict with notes describing the correct transform
- Field D: If a question exists, resolve it with an answer

### Verification checklist
- [ ] Learning record exists in `learning` table with correct content/scope/source
- [ ] Entity Knowledge context doc is rebuilt (check `context` table, subcategory=entity_knowledge)
- [ ] EK document includes the new MANDATORY phrasing and preamble
- [ ] Resolved question appears in EK under "Resolved Questions" section
- [ ] Context cache is invalidated (next generation would pick up changes)
